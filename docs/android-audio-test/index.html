Android Audio Diagnostics

Build v47 (locked-screen check) — run the tests in order, then tap Copy results at the bottom and paste the output back into chat.

What this answers: does Conductor's baked audio track keep playing — and stay on time — when this Android phone's screen is off in a pocket? Unlike iOS, Android usually keeps the page's JavaScript alive while audio plays, so a monitor logs objective evidence (media clock vs wall clock every 5s) alongside what you hear. Run in Chrome on the phone you'll actually use.

"Screen off" here = press the power button once so the screen goes dark. You do not need to unlock afterward until it's time to report.

Battery saver is the prime failure suspect. If a locked test fails, turn battery saver OFF (Settings → Battery → Battery Saver) and retry once before calling it real.

Environment

Tests

1. Bake sanity — screen on (~15s)
Runs the app's real pipeline (createTimelineAction → computeCueSchedule → bakeScheduleToWavBlob) on a tiny ~12s schedule and plays the result. If this browser can't bake at all, that's the finding by itself — it means the main app shows the "Keep your screen on" warning here instead of going pocket-ready.
—
2. Quick locked check (~35s)
Plays one continuous audio file with 7 rising beeps, 5s apart. Tap Run, and the moment you hear the FIRST beep, screen off (power button) and pocket it. Listen ~35s, then pull it out and tap what you heard. The monitor logs the media clock the whole time, so the log corroborates your ears.
—
3. THE GATE — real pipeline bake under lock (~3 min)
The Android mirror of the iOS ship-gate test. Bakes a real ~170s multi-cue timeline through the app's own pipeline and plays the one WAV from <audio>. Tap Run, hear the first beep, then screen off + pocket it for the full ~3 minutes, then report. Beeps come in short 3-2-1 clusters every 20s, rising in pitch. If it fails: battery saver OFF, retry once — only a failure with battery saver off counts as real.
—
4. Main-app spot check — record what happened (no audio)
After tests 1–3, run the real app once: open Conductor (footer must say build v47), Try the Demo → Start Practice → Go Live, note which banner appears, then lock + pocket for 3+ minutes and listen. Record both halves here so ONE paste carries the synthetic tests and the real-app result.
